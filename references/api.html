Parser¶

class r2x_reeds.ReEDSParser[source]¶

Bases: Plugin[ReEDSConfig]

Parser for ReEDS model data using the Plugin[ReEDSConfig] pattern.

This parser builds an infrasys.System from ReEDS model output through three main phases:

  1. Validation (on_validate()): - Presence of required datasets - Validity of configured solve and weather years - Loading and parsing of configuration assets

  2. Preparation (on_prepare()): - Parser context with configuration and defaults - Time indices and calendar mappings for the weather year - Generator datasets separated into variable and non-variable groups - Hydro capacity factor data for budget calculations - Reserve requirement configuration and costs

  3. Building (on_build()): - Regions from hierarchy data with regional attributes - Generators split into renewable (aggregated by tech-region) and non-renewable (with vintage) - Transmission interfaces and lines with bi-directional ratings - Loads with peak demand by region - Reserves by transmission region and type - Emissions as supplemental attributes on generators - Time series data (load profiles, capacity factors, reserve requirements, hydro budgets) - System metadata and description

Key Implementation Details¶

  • Renewable generators are aggregated by technology and region (no vintage)

  • Non-renewable generators retain vintage information

  • Reserve requirements are calculated dynamically based on wind capacity, solar capacity, and load data with configurable percentage contributions

  • Time series data is filtered to match configured weather years and solve years

  • Component caches are used during building for efficient cross-referencing

Examples

Build a ReEDS system from test data:

>>> from pathlib import Path
>>> from r2x_core import PluginContext, DataStore
>>> from r2x_reeds import ReEDSConfig, ReEDSParser
>>>
>>> config = ReEDSConfig(solve_year=2030, weather_year=2012, case_name="High_Renewable")
>>> store = DataStore(path=Path("tests/data/test_Pacific"))
>>> ctx = PluginContext(config=config, store=store)
>>> parser = ReEDSParser.from_context(ctx)
>>> result_ctx = parser.run()
>>> system = result_ctx.system

Notes

The parser uses internal caches for regions and generators to optimize cross-referencing during component construction. These caches are populated during the build phase and are used for all subsequent operations.

Initialize the ReEDS parser with internal state.

Note: Configuration and store are provided via from_context() class method.

Prepare and normalize configuration, time arrays, and component datasets.

Initializes internal caches and datasets required for component building: - Parser context with configuration and defaults - Time indices and calendar mappings for the weather year - Generator datasets separated into variable and non-variable groups - Hydro capacity factor data for budget calculations - Reserve requirement configuration and costs

Returns:

Ok() on successful preparation, Err() with error message if any step fails

Return type:

Result[None, str]

on_build()[source]¶

Build and return the complete infrasys.System.

Constructs all system components in dependency order: 1. Regions (required by all other components) 2. Generators (requires regions) 3. Transmission interfaces and lines 4. Loads (requires regions) 5. Reserves and reserve regions (requires transmission regions) 6. Emissions (requires generators) 7. Time series data (load profiles, capacity factors, reserve requirements, hydro budgets) 8. Post-processing (metadata and description)

Returns:

Ok(system) with the constructed power system, or Err() with error message

Return type:

Result[System, str]

Configuration¶

pydantic model r2x_reeds.ReEDSConfig[source]¶

Configuration for ReEDS model parser.

This configuration class defines all parameters needed to parse ReEDS model data, including year information and model-specific settings. Model-specific defaults and constants are loaded from configuration files in the config/ directory relative to this module.

Parameters:

  • solve_year (int | list[int]) – Model solve year(s) (e.g., 2030, [2030, 2040, 2050])

  • weather_year (int | list[int]) – Weather data year(s) used for time series profiles (e.g., 2012, [2007, 2012])

  • case_name (str, optional) – Name of the ReEDS case

  • scenario (str, optional) – Scenario identifier

Examples

Single year:

>>> config = ReEDSConfig(
...     solve_year=2030,
...     weather_year=2012,
...     case_name="High_Renewable",
... )

Multiple years:

>>> config = ReEDSConfig(
...     solve_year=[2030, 2040, 2050],
...     weather_year=[2007, 2012],
...     case_name="Multi_Year_Analysis",
... )

Load model defaults separately for use in parser:

>>> # Load defaults using the class method
>>> defaults = ReEDSConfig.load_config()
>>> excluded_techs = defaults.get("defaults", {}).get("excluded_techs", [])
>>>
>>> # Create config
>>> config = ReEDSConfig(
...     solve_year=2030,
...     weather_year=2012,
... )
